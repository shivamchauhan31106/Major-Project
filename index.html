How the inspection works

Authority & identity

Checks whether the address hides its real destination behind an "@" trick, a bare IP, or characters built to look like letters they aren't.

Brand impersonation

Looks for trusted names — banks, Apple, Google — sitting in a subdomain or path while the actual domain belongs to someone else entirely.

Domain composition

Flags risky extensions, excessive hyphens or digits, and structures that resemble auto-generated phishing domains.

Pressure language

Notices when a link stacks urgency words like "verify," "suspended," or "confirm" — a classic push toward a hasty click.

This is a heuristic first pass, not a verdict from a threat-intelligence database. A link can fail every check here and still be harmless, or pass every check and still be new enough to be dangerous. When in doubt, don't click — type the address yourself instead.
